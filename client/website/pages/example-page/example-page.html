The style guide is a reference page that shows how your application should look like. It starts with basic blocks like buttons, inputs, headings, tables, text, typography and colors and then continues with more complex UI elements like input groups, button groups, dropdowns, calendar and more.

As a general rule, any piece of UI that is going to be used more than one time needs to be added to the style guide.

It's certainly possible to go beyond that initial scope and extend the style guide to include larger scale patterns like layouts, or page/section template.

If you are looking for more examples of great style guides you can check styleguides.io.
